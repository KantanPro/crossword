クロスワードショートコードのテスト

修正内容の概要

クロスワードプラグインのショートコード処理で、empty()関数の使用による問題を修正しました。

問題: empty()は文字列"0"を空として扱うため、[crossword id="0"]の場合にデフォルトパズルが表示されていました。
修正: !isset($atts['id']) || $atts['id'] === '' で判定するように変更しました。

テストケース

1. ID指定のクロスワード

[crossword id="1"]
修正済み 期待される動作: パズルID 1のクロスワードが表示される

2. ID指定のクロスワード（ID 2）

[crossword id="2"]
修正済み 期待される動作: パズルID 2のクロスワードが表示される

3. ID指定のクロスワード（ID 0）

[crossword id="0"]
修正済み 期待される動作: パズルID 0のクロスワードが表示される（存在する場合）

4. デフォルトのクロスワード

[crossword]
動作中 期待される動作: デフォルトパズル（ID 1）のクロスワードが表示される

修正前後の比較

修正前（問題あり）

if (empty($atts['id'])) { $puzzle = $this->get_default_puzzle($atts['difficulty']); } else { $puzzle = $this->get_puzzle_by_id($atts['id']); }

問題点:

  • empty("0") → true （文字列"0"を空として扱う）
  • empty("2") → false （正しく動作）
  • empty("") → true （正しく動作）

修正後（改善済み）

if (!isset($atts['id']) || $atts['id'] === '') { $puzzle = $this->get_default_puzzle($atts['difficulty']); } else { $puzzle = $this->get_puzzle_by_id($atts['id']); }

改善点:

  • !isset($atts['id']) || $atts['id'] === '' → より正確な判定
  • 文字列"0"も正しくIDとして認識される
  • 空文字のみがデフォルトパズルの条件となる

テスト手順

  1. WordPressの管理画面で、このHTMLファイルの内容をページまたは投稿に貼り付ける
  2. 各ショートコードが正しく表示されるかを確認する
  3. 特に [crossword id="2"] が表示されることを確認する
  4. [crossword] でデフォルトパズルが表示されることを確認する

注意事項

重要: このHTMLファイルはテスト用です。実際のWordPressサイトでは、ショートコードを直接ページに記述してください。

例：

ここにクロスワード1が表示されます： [crossword id="1"] ここにクロスワード2が表示されます： [crossword id="2"] ここにデフォルトのクロスワードが表示されます： [crossword]
